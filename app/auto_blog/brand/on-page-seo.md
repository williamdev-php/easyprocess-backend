# THE COMPREHENSIVE ON-PAGE SEO CHECKLIST
## On-page SEO — do these on every page!
15 categories - 80+ items - the complete on-page SEO spec for blog posts and service pages. Technical SEO (sitemaps, robots.txt, Core Web Vitals) is covered separately.

**Every page-generation skill reads this before generating any page.**

### 1. HEAD & METADATA | What Google indexes first
- [x] Title tag — 50-60 chars, primary keyword near the start.
- [x] Meta description — 150-160 chars, keyword + benefit + soft CTA.
- [x] Canonical URL set to prevent duplicates.
- [x] Open Graph — og:title, og:description, og:image (1200x630), og:url, og:type.
- [x] Twitter Card — summary_large_image, title, description, image.
- [x] Language attribute on `<html>` (e.g. `lang="en"`).
- [x] Viewport meta tag for responsive rendering.
- [x] Favicon + apple-touch-icon.
- [x] Charset meta — `<meta charset="utf-8">`.

### 2. URL STRUCTURE | Clean, readable, keyword-forward
- [x] Short slug — under 60 chars.
- [x] Primary keyword in the slug.
- [x] Hyphens only — never underscores.
- [x] Lowercase only.
- [x] No stop words ("the", "a", "of") unless necessary.
- [x] Logical hierarchy — `/services/[slug]`, `/blog/[slug]`.

### 3. HEADINGS | Structure for skimmers & bots
- [x] Exactly one H1 per page, contains primary keyword.
- [x] Logical H2 -> H3 hierarchy — never skip levels.
- [x] H2s use supporting keywords + questions from the cluster.
- [x] No keyword stuffing — write naturally.

### 4. COPY & BODY | Answer the query, fast
- [x] First paragraph answers the query directly.
- [x] Short paragraphs — 2-4 sentences max.
- [x] Bucket brigades and transition phrases to maintain flow.
- [x] Bold key takeaways and important phrases.
- [x] Bulleted/numbered lists for scannable content.
- [x] 1500+ words for comprehensive coverage.

### 5. FAQ SECTION | Every blog post
- [x] 4-6 questions from SEMrush Questions tab + "People Also Ask".
- [x] Direct answers — 2-4 sentences each.
- [x] FAQ schema (JSON-LD) applied.

### 6. IMAGES | Every image is a ranking signal
- [x] Alt text describes image + keyword where natural.
- [x] Filenames — descriptive, hyphens, e.g. `emergency-plumber-toronto.webp`.
- [x] WebP, compressed under 200 KB.
- [x] Width/height attributes specified — prevents CLS.
- [x] Lazy loading (`loading="lazy"`) for below-fold.
- [x] Responsive srcset where needed.
- [x] Featured/hero image for social sharing.

### 7. INTERNAL LINKS
- [x] 3-5 internal links per post.
- [x] Link to related blog posts & relevant service pages.
- [x] Descriptive anchor text — never "click here" or "read more".
- [x] Contextually placed in body copy.
- [x] Breadcrumbs on every page.

### 8. EXTERNAL LINKS | Cite authority, don't hoard it
- [x] 2-3 external links to authoritative sources (.gov, .edu, major industry).
- [x] Relevant to the topic.
- [x] Open in new tab with `rel="noopener"`.
- [x] `rel="nofollow"` for sponsored links.

### 9. SCHEMA MARKUP | JSON-LD in <head>
- [x] Article schema on blog posts.
- [x] LocalBusiness schema — most specific subtype (Plumber, Dentist...).
- [x] Service schema on service pages.
- [x] FAQ schema wherever FAQ section exists.
- [x] BreadcrumbList schema on every page.
- [x] Organization schema site-wide.
- [x] Author/Person schema for bylines.

### 10. E-E-A-T SIGNALS | Experience - Expertise - Authority - Trust
- [x] Author byline with name on every blog post.
- [x] Author bio with credentials (years, qualifications).
- [x] Link to author's dedicated page.
- [x] Published date displayed.
- [x] "Last updated" date when refreshed.
- [x] Real stories, numbers, opinions from the business voice file.
- [x] Cite authoritative sources.
- [x] About page with full company credentials.
- [x] Contact page — real address, phone, hours.

### 11. ACCESSIBILITY | A11y signals = SEO signals
- [x] Semantic HTML5 — `<header>`, `<nav>`, `<main>`, `<article>`, `<footer>`.
- [x] ARIA labels on interactive elements where needed.
- [x] Color contrast meets WCAG AA (4.5:1 body text).
- [x] Focus indicators visible on interactive elements.
- [x] Alt text on all images (empty `alt=""` for decorative).
- [x] Descriptive link text.
- [x] Skip-to-content link for keyboard users.

### 12. MOBILE & RESPONSIVE | Mobile-first indexing
- [x] Responsive layout (Tailwind handles this).
- [x] Touch targets minimum 48x48 px.
- [x] Body font minimum 16 px.
- [x] No horizontal scroll at any viewport.
- [x] No intrusive interstitials.

### 13. SOCIAL PREVIEW | Shareable card
- [x] OG image optimized — 1200x630, under 1 MB.
- [x] Twitter Card image — 1200x600.
- [x] Compelling `og:description` — different from meta if valuable.

### 14. CONVERSION ELEMENTS | Capture the lead (SERVICE PAGES ONLY)
- [x] Primary CTA above the fold.
- [x] Phone number with click-to-call (`tel:`).
- [x] Multiple CTA placements throughout the page.
- [x] Trust signals — reviews, ratings, licenses, years.
- [x] Testimonials with names (photos where possible).
- [x] Service-area coverage listed.
- [x] Business hours displayed.
- [x] Physical address with embedded map.

### 15. LONG-FORM CONTENT | 1500+ word posts
- [x] Table of contents with anchor links at the top.
- [x] Jump links for each H2.
- [x] Back-to-top button.
